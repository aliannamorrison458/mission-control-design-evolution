# Mission Control Design Evolution

This repository tracks the evolution of AI-first operating system design for a one-person company, with OpenClaw Mission Control as the execution substrate.

## Goals

- Keep top-level architecture decisions versioned and auditable.
- Record rationale and trade-offs over time.
- Turn design artifacts into executable operating rules.

## Structure

- `docs/` — versioned design docs and ADR-like records.

## Initial Scope

- AI-first operating model
- Board Group / Dashboard planning
- Rolling scheduling and governance
